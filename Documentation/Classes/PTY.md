# PTY Class

The `PTY` class (`cs.xterm.PTY`) is an object-oriented wrapper for the `PTY4DPlugin`. It represents a pseudoterminal session acting as the backend engine for terminal applications inside 4D.

## Prerequisites

To use this class, ensure that the native plugin component is properly installed in your database `Plugins/` map:
* [PTYPlugin4D Repository](https://github.com/mesopelagique/PTYPlugin4D)

## Initialization

```4d
// cs.xterm.PTY.new($shell : Text; $cols : Integer; $rows : Integer; $cwd : Text)
var $pty : cs.xterm.PTY
$pty := cs.xterm.PTY.new("/bin/zsh"; 80; 24; Folder(fk database folder).path)
```

- **$shell**: The path of the shell executable you want to spawn (default: `/bin/zsh`).
- **$cols**: The number of columns for the terminal viewport (default: `80`).
- **$rows**: The number of rows for the terminal viewport (default: `24`).
- **$cwd**: The initial working directory for the spawned shell.

## Properties

- `id` (Integer): The internal cross-reference ID of the spawned PTY session.
- `shell` (Text): The path to the shell executable in use.
- `cols` (Integer): The current column width of the terminal.
- `rows` (Integer): The current row height of the terminal.
- `isOpen` (Boolean): Read-only method returning `True` if the session is actively open and successfully generated an ID.

## Methods

### `getStatus() : cs.xterm.PTYStatus`

Returns the status of the PTY session as a `cs.xterm.PTYStatus` object.
**Provides access to**:
- `pid` (Integer): The native process ID of the external process.
- `running` (Boolean): `True` if the process is currently running.
- `exitCode` (Integer): Returns the exit code if the process has terminated (otherwise `-1`).

### `write($text : Text) : cs.xterm.PTY`

Writes raw text into the PTY session standard input.
**Returns**: The `cs.xterm.PTY` instance to allow method chaining.

### `writeLine($text : Text) : cs.xterm.PTY`

Writes raw text appended with a standard newline (`\n`) into the PTY session.
**Returns**: The `cs.xterm.PTY` instance to allow method chaining.

### `read($bufferSize : Integer; $timeoutMs : Integer) : Text`

Reads incoming data generated by the PTY session.
- **$bufferSize**: The maximum number of bytes to read per call (default `65536`).
- **$timeoutMs**: The maximum duration to pause and wait for incoming data before timing out (default `2000`).

### `readAll($timeoutMs : Integer) : Text`

Continuously reads incoming data from the PTY session until the buffer is empty. Pauses up to `$timeoutMs` (default `500`) waiting for each incoming buffer chunk.

### `resize($cols : Integer; $rows : Integer) : Integer`

Resizes the pseudo-terminal native window to match new dimensions. Typically called when a UI element like a Web Area scales or undergoes a window dimension drag.

### `sendSignal($signal : Integer) : Integer`

Sends an explicit Unix POSIX signal to the underlying child process.

### `interrupt() : Integer`

Sends a `SIGINT` (Signal 2) interrupt to the process. This behaves exactly the same as initiating a manual interrupt using `Ctrl+C` over a standard terminal environment, hitting the entire process group.

### `close()`

Immediately shuts down the PTY session and drops underlying daemon resources to prevent zombie processes or resource leaks. Note: Calling `close` multiple times is handled securely empty without side-effects.
